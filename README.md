Название и назначение
Library Management API — веб-сервис для управления каталогом книг в библиотеке. API позволяет добавлять, искать, обновлять и удалять книги, а также управлять их доступностью.

Решаемая задача и целевая аудитория
Задача: Создание простого и эффективного REST API для управления коллекцией книг в цифровой библиотеке.

Целевая аудитория:

Библиотекари — для управления каталогом книг

Образовательные учреждения — для организации библиотечных фондов

Разработчики — для изучения ASP.NET Core и REST API

Студенты — в качестве учебного проекта

Основные функции
1. Управление книгами (CRUD)
POST /api/books — добавление новой книги

GET /api/books — получение всех книг

GET /api/books/{id} — получение книги по ID

PUT /api/books/{id} — обновление данных книги

DELETE /api/books/{id} — удаление книги из каталога

2. Поиск и фильтрация
GET /api/books/available — только доступные для выдачи книги

GET /api/books/genre/{genre} — книги по жанру

GET /api/books/author/{author} — книги по автору

POST /api/books/search — расширенный поиск с фильтрами

3. Управление доступностью
POST /api/books/{id}/borrow — взять книгу (отметить как недоступную)

POST /api/books/{id}/return — вернуть книгу (отметить как доступную)

4. Аналитика и статистика
GET /api/books/statistics — статистика библиотеки

GET /api/books/genres — список всех жанров

GET /api/books/authors — список всех авторов

Архитектурные особенности
Язык и фреймворк: C#, ASP.NET Core 8

Архитектура: Clean Architecture с разделением на слои

Хранение данных: In-memory коллекция (легко заменить на Entity Framework + SQL Server/PostgreSQL)

Валидация: Data Annotations для проверки входных данных

Документация: Swagger UI с автоматической генерацией

5.Требования:
.NET 8 SDK

IDE (Visual Studio 2022+, VS Code, Rider)

Браузер для тестирования

(Опционально) Postman/Insomnia для тестирования API

Обработка ошибок: Глобальный middleware для обработки исключений

Логирование: Встроенная система логирования ASP.NET Core
